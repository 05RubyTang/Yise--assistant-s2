### 4. 图鉴赛季精灵过滤

**需求**：图鉴中方案的 shinies 跟着赛季走

**问题**：
- S2 赛季查看火系方案 → 显示所有赛季精灵（S1+S2）
- S2 赛季查看萌系方案 → 显示 S1 的治愈兔、大耳帽兜

**解决**：修改 `src/pages/Collection.jsx` 的 PlanInfo 组件

**核心改动**：
```javascript
// 1. 添加赛季过滤函数
const filterShiniesBySeason = (shinies) => {
  if (!shinies || !currentSeason) return shinies || [];
  const seasonSpirits = getSpiritsBySeason(currentSeason);
  return shinies.filter(name => seasonSpirits.all.includes(name));
};

// 2. 过滤 visibleShinies
const rawShinies = isNoShiny ? (plan.poolShinies || []) : (plan.shinies || []);
const visibleShinies = filterShiniesBySeason(rawShinies);

// 3. 传入 currentSeason
<PlanInfo plan={plan} currentSeason={currentSeason} />
```

**效果**：
- S1 赛季 - 火系方案：只显示治愈兔、火红尾、柴渣虫
- S2 赛季 - 火系方案：只显示灵狐、烟花伯爵、爆焰喷喷
- S2 赛季 - 萌系方案：只显示菊花梨、加油海葵

**文档**：`图鉴赛季精灵过滤修改报告.md`

```
├── seasons/
│   ├── s1Plans.js  # S1 独立管理
│   └── s2Plans.js  # S2 独立管理
├── common/
│   └── getPlanFruitsArray.js  # 通用函数
└── plans.js  # 统一入口
```

### 2. 双属性精灵处理

- **刷取时**：只计第一属性池进度
- **出货时**：第一属性和第二属性池都可能出货
- **数据策略**：同时归入两个属性池的 shinies

**示例**：加油海葵（水+萌）
- 归入水系方案 shinies（第一属性）
- 归入萌系方案 shinies（第二属性）

### 3. 赛季隔离显示

- 方案数据包含所有赛季精灵（数据层）
- 显示时根据赛季过滤（显示层）
