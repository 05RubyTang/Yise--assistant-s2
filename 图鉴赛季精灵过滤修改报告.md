# 图鉴赛季精灵过滤修改报告

> **修改时间**：2026-05-16  
> **修改目标**：图鉴中方案的 shinies 列表跟着赛季走，只显示当前赛季的精灵

---

## 📋 问题描述

### 修改前的问题

在图鉴页面中，方案的 shinies 列表会显示所有精灵，包括跨赛季的精灵。

**示例**：
- 用户切换到 S2 赛季
- 查看「火系方案」的详情
- shinies 显示：`['治愈兔', '火红尾', '柴渣虫', '灵狐', '烟花伯爵', '爆焰喷喷']`
- **问题**：治愈兔、火红尾、柴渣虫 是 S1 精灵，不应该在 S2 赛季显示

### 期望行为

- 在 S1 赛季：只显示 S1 精灵（治愈兔、火红尾、柴渣虫）
- 在 S2 赛季：只显示 S2 精灵（灵狐、烟花伯爵、爆焰喷喷）

#### 2. 添加赛季过滤函数

**位置**：第 160-164 行（新增）

**代码**：
```javascript
// 根据当前赛季过滤精灵列表
const filterShiniesBySeason = (shinies) => {
  if (!shinies || !currentSeason) return shinies || [];
  const seasonSpirits = getSpiritsBySeason(currentSeason);
  return shinies.filter(name => seasonSpirits.all.includes(name));
};
```

**说明**：
- 使用已有的 `getSpiritsBySeason` 函数获取当前赛季的所有精灵
- 过滤 shinies 数组，只保留属于当前赛季的精灵
- 如果 currentSeason 未定义，返回原始列表（向后兼容）

## 🎯 设计原则

### 1. 赛季隔离

- 用户在 S1 赛季只看到 S1 精灵
- 用户在 S2 赛季只看到 S2 精灵
- 避免跨赛季信息干扰

### 2. 数据一致性

- 方案的 shinies 数据不变（仍包含所有赛季精灵）
- 只在显示层进行过滤
- 保证三池保底计数等逻辑仍正确运行

### 3. 向后兼容

- filterShiniesBySeason 会检查 currentSeason 是否存在
- 如果不存在，返回原始列表
- 确保旧数据不受影响
