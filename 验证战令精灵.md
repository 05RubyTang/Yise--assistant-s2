# 战令精灵数据验证

## 问题描述

图鉴中缺少 S2 战令异色精灵（雪怪、爆焰喷喷），原因是这两个精灵不在 `ALL_SHINIES` 数组中。

## 根本原因

`ALL_SHINIES` 是从 `PLANS` 数组中提取所有方案的 `shinies` 字段生成的：

```javascript
export const ALL_SHINIES = [...new Set(PLANS.flatMap(p => p.shinies))].filter(Boolean);
```

而 s2Plans.js 中只有 16 个方案（8个赛季常驻 + 8个赛季奇遇），缺少战令异色的方案数据。

## 解决方案

在 `src/data/seasons/s2Plans.js` 中添加战令异色的方案数据：

### 新增方案（2个）

1. **雪怪**
   - ID: `s2_battlepass_snowman`
   - 属性: 冰系
   - 类型: 战令异色（battlePass: true）
   - 解锁方式: 购买战令礼包

2. **爆焰喷喷**
   - ID: `s2_battlepass_blast_flame`
   - 属性: 火系（第二属性：龙系）
   - 类型: 战令异色（battlePass: true）
   - 解锁方式: 购买战令礼包

添加方案数据后：

## 数据流向

```
s2Plans.js
  ↓
S2_PLANS 数组（18个方案）
  ↓
PLANS = [...S1_PLANS, ...S2_PLANS]（51个方案）
  ↓
ALL_SHINIES = PLANS.flatMap(p => p.shinies)（37个精灵）
  ↓
store.jsx buildDefaultState()
  ↓
spirits = { '雪怪': {...}, '爆焰喷喷': {...}, ... }
  ↓
图鉴页面正常显示所有精灵
```

## 注意事项

1. **战令精灵不通过刷取获得**，而是购买战令礼包获得专属果实
2. **图鉴中的标记方式**：用户手动点击标记为已获得
3. **不影响三池保底**：战令精灵单刷只进家族池，不计入属性池/世界池
4. **果实图片**：需要补充 `public/fruits/` 目录下的战令果实图片（如有）
